## ИНДИВИДУАЛЬНЫЕ ПРАВКИ ДУДИН АРТЕМ 501690

### 1) Добаление возможности сохранения графиков в отдельную папку, создание этой папки.

Эта правка расширит функционал программы. У пользователя появится возможность сохранить график для 
дальнейшего изучения или показа в других местах. На мой взгляд, это очень важное и полезное изменение

Благодаря существованию файла utils, можно довольно легко добавить возможность сохранения графика. 
Для этого достаточно добавить функцию save_plot, которая будет сохранять графики в папку и будет 
вызываться внутри функции show_plot.
Это позволит не менять каждый модуль, добавляя вызов новой фунции, достаточно будет изменений в utils. 
(См. utils, комментарии капсом)

### 2) Учет фидбека к проекту 

Действительно, немного не согласовали работу, получилось так, что модули dynamic_sales 
и revenue_time сильно дублируют друг дрга. 

Поэтому просто удаляем модуль dynamic_sales, а также его импорты в interface и init, удаляем пункт 
"Динамика продаж за период" из interface и соответствующим образом подгоняем под это цикл в 
фунцкии get_user_choise. 

### 3) Добавление новых графиков

Новые графики - новые виды анализа, новые данные для пользователя. 
Добавление новых графиков расширит программу, сделает так, 
что она будет еще более полно передавать данные о продажах из базы данных.
Воспользуемся тем, что модульная структура позволяет добавлять сколько 
угодно новых графиков без больших тркудностей

Вот ход моих действий:

1) Создать новые модули в папке modules, написать код
2) Добавить импорты в init
3) В interface дополнить функцию get user choise: добавить новые пункты в
   меню, добавить вызов новых модулей в цикл.

### Новые модули:
1) payment_analysis - создает круговую диаграмму, отображающую процентноее соотношение
   частоты использования разных способов платежей.
2) profit_top - создает столбчатую диаграмму топа в сооствествии с выбором пользоваиеля:
   либо наболее прибыльных подкатегорий товаров, либо наименее подкатегорий прибыльных.
